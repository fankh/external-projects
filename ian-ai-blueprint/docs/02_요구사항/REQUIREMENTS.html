프로젝트 개요
(Project Overview)
목적 (Objective)

본 요구사항 정의서는 AI 기반 2D 도면 생성 시스템(ian-ai-blueprint) 사업의 기능 요구사항과 비기능 요구사항을 정의합니다.

범위 (Scope)

웹 브라우저에서 기존 CAD 파일(DXF·DWG·IFC)을 업로드하여 2D 도면으로 렌더링하고, AI 프롬프트로 도면을 생성하며, 결과를 DXF로 내보내는 경량 웹 CAD 시스템의 구축을 범위로 합니다. 모든 임포터와 AI 생성기는 하나의 정규화된 DrawingDocument JSON(line/polyline/circle/arc/text 엔티티 + 레이어)을 생성하고, 프론트엔드는 이 형식만 렌더링합니다.

주요 목표 (Key Goals)
  • CAD 파일(DXF·DWG·IFC)을 전용 소프트웨어 없이 웹에서 2D 도면으로 열람
  • 자연어 프롬프트 기반 AI 도면 초안 자동 생성으로 설계 생산성 향상
  • 표준 DXF(AutoCAD R2010) 내보내기로 기존 CAD 도구와의 상호운용성 확보

FR-001 CAD 파일 업로드 (F-001) DXF·DWG·IFC 파일을 드래그앤드롭으로 업로드할 수 있어야 하며, 확장자 검증과 최대 50MB 크기 제한을 적용한다. High 테스트
FR-002 2D 도면 렌더링 (F-002) 업로드된 도면을 SVG로 렌더링하고 팬/줌, 레이어별 색상 표시, CAD Y-up 좌표계 변환을 지원해야 한다. High 테스트
FR-003 AI 도면 생성 (F-003) 자연어 프롬프트를 입력하면 Claude API를 통해 2D 평면도(DrawingDocument JSON)를 자동 생성해야 한다. API 키 미설정 시 스텁 생성기로 동작한다. High 테스트
FR-004 DXF 내보내기 (F-004) 렌더링된 도면을 AutoCAD R2010 DXF 파일로 변환하여 다운로드할 수 있어야 하며, 내보낸 파일은 재업로드 시 동일하게 렌더링되어야 한다. High 테스트
FR-005 레이어 관리 (F-005) 도면의 레이어 목록을 표시하고 레이어별 표시/숨김 토글을 제공해야 한다. Medium 테스트
FR-006 IFC 층 선택 (F-006) IFC 파일에서 storeyIndex 파라미터로 층을 선택하여 해당 층의 2D 평면을 추출해야 하며, 범위를 벗어난 storeyIndex 요청 시 명확한 오류를 반환해야 한다. Medium 테스트

제약 사항
(Constraints)
  • 기술 제약: DWG 파일은 직접 파싱하지 않고 ODA File Converter를 통해 DXF로 변환하여 처리하며, 변환기 미설치 시 DWG 기능은 제공되지 않음(501 안내). AI 도면 생성 품질은 Claude API에 의존
  • 규정 제약: 업로드되는 CAD 도면의 저작권 및 기밀 유지 준수, AI API 전송 데이터에 대한 고객사 보안 정책 준수

용어 정의
(Glossary)
용어 (Term) 정의 (Definition)
DXF (Drawing Exchange Format) AutoCAD 도면 교환용 개방형 파일 포맷. 본 시스템은 ezdxf로 파싱하고 AutoCAD R2010 DXF로 내보냄
DWG AutoCAD 고유 바이너리 도면 포맷. ODA File Converter를 통해 DXF로 변환 후 처리
IFC (Industry Foundation Classes) BIM 데이터 교환 표준 포맷. ifcopenshell로 층(storey)별 2D 평면을 추출
DrawingDocument 모든 임포터와 AI 생성기가 생성하는 정규화된 도면 JSON 스키마 (line/polyline/circle/arc/text 엔티티 + 레이어)
storeyIndex IFC 파일에서 추출할 층을 지정하는 인덱스 파라미터
스텁 생성기 (Stub Generator) Claude API 키 미설정 시 AI 도면 생성 요청에 대해 예시 평면도를 반환하는 대체 생성기
