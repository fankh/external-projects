B-06

A안 vs B안 — 권고

디자인 결정을 위한 비교

A안 — Modern SaaS (v0.1)

적합: 경영 Dashboard · 로그인 · 모바일 · 영업 데모
강점: 첫인상·브랜드 전달·신규 세대 사용자
약점: 밀도 부족(같은 높이에 ~14행) · MDI 부재 · 레거시 사용자의 학습 비용 — Set-up·ERP 실무 화면에는 부적합

B안 — Dense Enterprise ★ 권고

적합: CPQ·코드 Set-up·발주·업무함·Run 등 일 8시간 화면 전부
강점: 밀도(~24행) · F-key 완결 · MDI 멀티태스킹 · 더존/SAP 근육 기억 호환 → 교육 비용 최소화 = SI 검수 리스크 감소
약점: 첫인상이 보수적 — 데모용 화면은 A안 요소 차용으로 보완
최종 권고 — 하이브리드, B 우세 배분
① 업무 화면(운영 8h) = B안: 22px 컨트롤·그리드 문법·MDI·F-key·조회밴드 — 97메뉴 중 약 85개
② 브랜드 접점 = A안 유지: 로그인 · Dashboard(경영 열람) · 모바일 App · Run 완료 요약(데모의 하이라이트만 카드형)
③ 공통 불변: Run 그린 예약색 · 상태 칩(색+형태) · tabular-nums · 포커스 링 — 두 안이 같은 토큰 뿌리를 공유하므로 혼용 비용 낮음
④ 다음 단계: B안 문법으로 W-04~W-24 잔여 화면 순차 전환 + 사용자 인터뷰(더존/SAP 경력자 3인)로 F-key 배열 검증
