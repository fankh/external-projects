===== NOTES for SLIDE 5 =====
1, 업무 흐름 (제품 선정)
Project 검색 – 장비 일람표 확인 – 제품의 기본 Arrangement Code 작성 – 필요에 따라 해당 이미지를 배치하여 제품의 구성 – 제품 Code 생성
2, 이미지 재배치 방법 :
정해진 이미지 표의 여러 이미지를 임의의 장소로 드래그하여 제품 구성을 조합
조합된 구성품을 이동, 삭제 및 구성 모듈의 분할 통합
조합된 이미지(도면)의 치수를 DB에서 가져와 표시
임의의 기준 치수를 제시 하였을 때 하부의 자료는 임의의 치수를 반영
화면과 도면의 비율이 자동 조절
이미지(도면)을 3각법으로 선택 하여 표시 (각 6면 및 전체 도면)
모듈 이미지를 000하여 제품의 속성 확인
모듈 이미지를 더블클릭하여 제품 속성(기술, 물성, 세부 치수)이 표기되는, 새로운 각 모듈 세부선정 UI 창을 띄워 제품 속성의 정의 수정
최종 정리된 사양으로 도면, 부품표, 기술자료 출력(File 또는 Print)
3. Arrangement Code
표준이 되는 기본 구성 조합을 DB로 준비
구성 조건은 “Product”의 Hierarchy의 Data로 존재
별도의 구성 기준표 활용
