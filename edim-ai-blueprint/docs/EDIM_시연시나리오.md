# EDIM 시연 시나리오

> 대상: 고객사 경영진·현업 실무자 / 소요: 약 10분 (질의응답 별도) / 갱신: 2026-07-09

---

## 1. 개요

본 문서는 EDIM Tool System 데모 사이트(https://edim.seekerslab.com)를 이용한 표준 시연 절차서다.
시연의 핵심 메시지는 세 가지다.

1. **하나의 코드 체계(RCCS)** 로 CPQ → PLM → ERP 가 실데이터로 관통한다 — 화면 간 이동이 아니라 **동일 DB(PostgreSQL 16) 위의 단일 흐름**임을 보인다.
2. **설계 자동화가 실제로 동작한다** — Macro 수식 평가, 파라메트릭 CAD 재작도, Run 파이프라인의 실산출물(DXF·PDF·XLSX)을 그 자리에서 내려받아 연다.
3. **통제가 내장되어 있다** — Running Test 게이트, 승인 워크플로, 권한(RBAC), 이력.

## 2. 사전 준비 체크리스트

| # | 점검 | 방법 | 기대 |
|---|------|------|------|
| 1 | 백엔드 정상 | 로그인 후 우하단 상태바 | `DB: EDIM-PRD (PG16)` (— `DB: MOCK` 이면 백엔드 점검) |
| 2 | 계정 | `edim / edim` | ADMIN 권한 (승인·잠금해제 데모 가능) |
| 3 | 브라우저 | Chrome/Edge 최신, 새로고침 1회 | 최신 번들 로드 (index.html no-cache 적용됨) |
| 4 | 화면 해상도 | 1440×900 이상 권장 | MDI·우측 패널 겹침 없음 |
| 5 | 열린 탭 정리 | 탭 전체 닫기 후 시작 | 탭은 localStorage 에 영속되므로 이전 시연 잔재 제거 |

접속 URL 은 모듈 어디로 들어가도 된다: `/cpq` `/plm` `/code` `/erp` `/toolbox` `/common`.

## 3. 타임라인 요약

| 분 | 코너 | 화면 | 핵심 메시지 |
|----|------|------|------------|

## 4. 단계별 상세

### 4-1. 로그인과 셸

| 조작 | 기대 결과 |
|------|----------|
| https://edim.seekerslab.com/cpq 접속 | dense 로그인 화면 |
| 사번 `edim` / 비밀번호 `edim` → 로그인 | MDI 셸 진입, 타이틀바 `Micron #7 (Pre-Sales) · YS.Gang [ADMIN]` |
| 우하단 상태바 확인 | `DB: EDIM-PRD (PG16)` |
| 타이틀바 `KO ▾` 클릭 → EN 선택 | 메뉴바·F-key·상태바 즉시 영어 전환 (sys_translation 실데이터) — 다시 KO 로 복귀 |

**멘트**: "화면은 전량 실코드이며 지금 보시는 데이터는 서버 PostgreSQL 실DB입니다. 우하단 표시가 그 증거입니다."

### 4-2. CPQ 제품선정

화면: **CPQ > 제품선정 C-1**

| 조작 | 기대 결과 |
|------|----------|
| KOF 슬롯에서 임의 항목 선택 변경 | 하단 BOM 그리드 **실시간 재전개** — 루트 `KDP 1-21-13-15` |
| BOM 행 더블클릭 | **코드 상세 탭** — 도면·단가 이력·Referencers·승인 이력 |
| 캔버스 우측 `CAD` 토글 | 서버 작도 구성 배치 도면 (ezdxf 실도면, 전체 치수 4504×3254) |
| 휠·드래그로 줌/팬, 더블클릭 = 맞춤 | CAD 탐색 동작 |
| `⬇ DXF` | 구성도 DXF 파일 다운로드 — AutoCAD 등에서 열림 |

**멘트**: "선택이 바뀌면 BOM 이 다시 전개됩니다. 이건 화면 연출이 아니라 서버의 재귀 CTE 가 code_relationship 을 실시간으로 다시 계산한 결과입니다."

### 4-3. PLM 설계 편집기

화면: **PLM > Design Editor S-4-1-1** — 진입하면 **기본이 CAD 모드** (서버 작도 부품도)

| 조작 | 기대 결과 |
|------|----------|
| 진입 직후 | 부품도 CAD 자동 로드 — 케이싱·임펠러·샤프트·베어링·치수선 `A = 670` |
| 휠 줌·드래그 팬 | 커서 기준 확대·이동 (우상단 %·＋/－/⌂) |
| 우측 치수 그리드 `A` 더블클릭 → `700` 입력 | 셀 값 변경 |
| `Run F9` | Macro 엔진 실평가 — `B = 756`, `K = 1134` 자동 계산, **CAD 도면 즉시 재작도** (치수선 갱신) |
| `DXF 내보내기` | 현재 치수가 반영된 DXF 다운로드 |

**멘트**: "치수 하나를 바꾸면 종속 치수가 수식(Macro)으로 재계산되고 도면이 다시 그려집니다. 이 도면은 Run 이 만드는 제작도면과 동일한 정본입니다."

### 4-4. EDIM Run 실행

화면: **CPQ > EDIM Run**

| 조작 | 기대 결과 |
|------|----------|
| `Run F9` | 단계 진행: BOM 전개 → 치수 Macro 평가 → 제작도면 DXF → 단가 → 견적서 PDF·BOM XLSX |
| 산출물 그리드 `미리보기` (DXF 행) | **CAD 뷰어 탭** — 레이어 토글·줌/팬 |
| `⬇` 다운로드 (PDF 행) | 견적서 PDF — 한글 임베드·CONFIDENTIAL 워터마크·실 BOM 합계 |
| 공통 > Project Folder 열기 | 방금 산출물이 폴더에 자동 등록됨 |

**멘트**: "Run 은 목업이 아니라 실제 파이프라인입니다. 방금 만들어진 DXF·PDF·XLSX 를 지금 내려받아 열 수 있고, MinIO 저장소와 문서 대장에 자동 등재됩니다."

### 4-5. 코드 통제 게이트

화면: **Code Set-up > Code Relationship S-1-4**

| 조작 | 기대 결과 |
|------|----------|
| 승인 버튼 확인 | **비활성** — Running Test 미통과 상태 |
| `Running Test` 실행 | 전개 검증 통과 — `KDP 1-21-13-15` |
| 승인 버튼 재확인 | **활성화** — 게이트 해제 |

**멘트**: "관계 변경은 테스트를 통과해야 승인 요청이 가능합니다. 설계 실수가 코드 체계로 유입되는 것을 시스템이 막습니다."

### 4-6. ERP 와 승인 워크플로

| 조작 | 기대 결과 |
|------|----------|
| **ERP > Dashboard M-14-4** | KPI·공정 흐름 실집계 (erp_process_event) |
| 이상경고 더블클릭 → 사유 입력 → 완료 | 경고 해제, 후행 공정 이벤트 생성 |
| **ERP > 단가 관리 M-12-5** — `FDV-480` resolve | 우선순위 ①적용가→②구매가→③재고가→④견적가 시뮬레이션 → `450,000 (견적·효성)` |
| **공통 > 승인함 M-15-2** — 대기 건 승인 | 대상 상태 전이 + 이력 기록 + **타이틀바 벨 알림** 생성 |

**멘트**: "승인 한 건이 처리되면 요청자에게 알림이 가고 sys_history 에 남습니다. 누가 언제 무엇을 바꿨는지 전부 추적됩니다."

### 4-7. Toolbox 와 마무리

| 조작 | 기대 결과 |
|------|----------|
| **Toolbox > Macro Studio S-2-2** — Test Run | 수식 실평가 `786` → TESTED 게이트 통과 |
| **Toolbox > UI Designer S-2-1** — AI 초안 | 화면 초안 자동 생성 (현재 샘플 모드 — API 키 설정 시 Claude 실연동) |
| **공통 > Mobile 미리보기 M-16** | 모바일 3대 프리뷰 |
| 브라우저 새로고침 | **열린 탭 전부 복원** (localStorage 영속) |

**멘트**: "현장 엔지니어가 쓰는 수식 도구까지 같은 엔진입니다. AI 연동은 코드가 완성되어 있고 키만 넣으면 활성화됩니다."

## 5. 예상 질의응답

| 질문 | 답변 요지 |
|------|----------|
| 데이터가 진짜인가? | 서버 PostgreSQL 16, 54테이블. 상태바 `DB: EDIM-PRD`. MOCK 폴백은 백엔드 중단 시 데모 연속성용이며 표시로 구분됨 |
| DWG 도 되나? | DXF 는 완료. DWG 는 ODA 변환기 플러그블 구조로 준비 완료 — 라이선스 확정 시 환경변수 설정만 |
| AI 는? | Macro 생성·UI 초안 API 구현·배포 완료, Claude 키 설정 시 즉시 실연동 (현재 샘플 모드) |
| 권한은? | HMAC 토큰 + 레벨 RBAC — 쓰기=SETUP 이상, 관리 작업=ADMIN. GENERAL 계정은 403 |
| 백업·배포는? | 매일 03:20 pg_dump+MinIO 백업(7일 보존), git push 후 2분 내 자동 배포 |
| 다국어는? | KO/EN/JA/ZH 런타임 전환 — 번역은 sys_translation 테이블 관리 |

## 6. 트러블슈팅

| 증상 | 원인 | 조치 |
|------|------|------|
| 상태바 `DB: MOCK` | 백엔드 재기동 창(배포 직후 수십 초) 또는 장애 | 30초 후 새로고침. 지속 시 `ssh edim-server` → `docker compose ps` |
| 로그인 후 화면 요소가 예전 모습 | 브라우저 캐시 | 새로고침 (index.html no-cache 적용 — 1회면 충분) |
| Run 산출물 미리보기 안 열림 | 백엔드 CAD 파싱 불가 상태 | MOCK 여부 확인 → 백엔드 회복 후 재시도 |
| 탭이 너무 많이 복원됨 | 이전 세션 탭 영속 | 탭 × 로 정리 (Run 탭은 영속 제외) |
