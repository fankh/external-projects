## 검증 방법론 (전 사이클 공통)

1. 전체 `npm run build`(컴파일+타입) + `py_compile`/`pytest` 게이트 → 커밋·푸시
2. edim-autodeploy(health-gate·자동 롤백) 배포 대기
3. edimsol.com 라이브 E2E (Playwright + API) — 생성 데이터는 매 사이클 삭제·원복
4. docs/EDIM_원본PPT_기능대조.md 체크오프 + EDIM_진행현황.md changelog
