문의하신 항목별로 유사 구축 사례 기준의 개략적인 범위를 아래와 같이 공유드립니다. 실제 범위·요구사항 확정 시 달라질 수 있는 참고치입니다.

## 1. 구축 단계별 예상 기간

| 단계 | 주요 작업 | 예상 기간 |
|------|-----------|-----------|
| 데이터 준비 | 정상·이상거래 데이터 확보/정비, 라벨 검증, Rule 분석, Feature 정의 | 2~3개월 |
| 환경 준비(솔루션 연계) | 인프라 구성, 데이터 송수신·탐지결과 연계 개발, 연계 테스트 | 1.5~2개월 |
| 시범운영(Pilot) | 모델 학습·검증, 병행 운영(기존 Rule 대비 성능 비교), 임계값 조정 | 2~3개월 |
| 실운영 전환 | 운영 전환, 모니터링·검토 프로세스 정착, 안정화 | 1~2개월 |

- 전체 약 **7~10개월** (데이터 준비와 환경 준비 일부 병행 시 6~8개월 수준까지 단축 가능)
- 전제: 학습에 필요한 이상거래 판정(Label) 데이터가 이미 축적되어 있는 경우 기준. 라벨 정비가 추가로 필요하면 데이터 준비 기간이 늘어납니다.

## 4. 주요 고려사항 및 선행 준비사항

1. **라벨(최종 판정) 데이터 품질** — 지도학습 성능은 판정 데이터의 축적 기간·표본 수·판정 일관성이 좌우합니다. 최근 2~3년치 판정 이력의 기준 일관성 점검을 선행하시길 권장합니다. 이상거래 표본이 적을 경우 비지도(이상탐지) 중심으로 시작해 라벨 축적 후 지도학습을 강화하는 단계적 접근이 현실적입니다.
2. **범위의 단계화** — 데이터가 갖춰진 해외송금부터 1차 적용 후 수출·수입으로 확장하는 방식이 위험과 기간 관리에 유리합니다.
3. **차세대·데이터 구조 안정화와의 일정 정합** — 자료에 언급하신 대로 데이터 구조 확정 이후 본격 학습이 가능하므로, 차세대 전환 일정과의 선후 관계를 사업 착수 전에 확정할 필요가 있습니다.
4. **기존 AI-FDS와의 역할 분담** — 중복 탐지·알림 처리 기준, 케이스 관리 통합 여부를 초기에 정리해야 운영 혼선을 줄일 수 있습니다.
5. **규제·심사 대응** — 전자금융감독규정, 자금세탁방지(FIU 보고)와의 연계, 신용정보법상 데이터 활용 범위, 망분리 환경에서의 학습·운영 환경 구성, 모델 설명가능성(XAI) 확보가 금융권 공통 요구사항입니다.
6. **오탐 관리 체계** — 실시간 알림량이 담당자 검토 가용량을 넘지 않도록 임계값·우선순위 정책과 검토 프로세스를 Pilot 단계에서 함께 설계해야 합니다.
