## 슬라이드 1 — 단계적 구축 방향

4단계 구성: **데이터 준비 → 솔루션 연계 → 업무 적용 → 운영 체계**

| 단계 | 세부 내용 |
|------|-----------|
| 데이터 확보·정비 | 정상거래내역, 이상거래이력, 현행 탐지 Rule 확보 및 정비 |
| AI 탐지 솔루션 연계 | 데이터 송수신, 위험도·탐지 결과 연계 |
| 실시간 탐지·검토·조치 | 실시간 모니터링, 알림, 담당자 검토·조회, 조치 및 결과 관리 |
| 운영체계·성능 고도화 | 탐지 결과·Rule 관리(추가·수정·폐지), 모델 성능 관리, 지속 개선 |

추진 일정 구상: 데이터 준비 → 환경 준비(구성·연계 테스트) → **시범운영(파일럿·모델 검증, 약 2~3개월)** → 실운영(운영전환·안정화)

문의자 측 인식(슬라이드 명기): AI 학습은 최종 확정된 데이터 구조·업무 프로세스 기반으로 수행되므로, **차세대 데이터·프로세스 안정화 이후 AI 고도화를 추진**하는 것이 데이터 품질·모델 정확성에 효과적. → 차세대 시스템 전환을 병행 중이거나 앞두고 있음을 시사.

## 슬라이드 2 — AI 학습 및 모델 적용 구상

- **학습 방식**: 정상 거래 데이터 → 비지도학습, 최종 이상거래 판정(Label) 데이터 → 지도학습 병행
- **Feature 구상**: 현행 탐지 Rule 분석 → Rule 기반 Feature 후보 도출 + 고객·거래·국가·금액·사유코드 특성 정의
  - 예시: 고객/업종/국가별 평균 거래금액, 기간별 거래빈도(1·3개월), 거래목적(사유코드), 상대 국가 변경 빈도, 거래금액 증감률
- **보유 데이터 소스(당행 시스템 명시)**: 해외이상송금점검내역, 해외이상송금거래내역, 외화송금제어등록, FISS해외송금계좌기본, 고객실적내역, 고객거래내역, 당행 해외송금 이상거래 모니터링 Rule(실시간 운영 중)
- **협업 전제**: 당행 AI부서 또는 AI 전문업체와 협업 필요, 외부 업체 AI 기술·솔루션 검토 중

## 시사점 (회신 자료 작성 시 반영)

1. **구상안이 이미 구체적** — 데이터 소스 테이블명까지 특정되어 있고 해외송금 이상거래 Rule 을 실시간 운영 중. 탐지체계 신규 도입이 아니라 기존 Rule 기반 체계의 AI 전환·고도화에 가깝다. 회신도 "제로베이스 구축"이 아닌 "Rule→AI 전환" 사례 기준으로 잡는 것이 적절.
2. **범위는 해외송금 중심으로 시작** — 문의 본문은 송금·수출·수입 3개 업무를 언급하나 슬라이드 데이터 소스는 해외송금 계열만 등장. 1차 범위(송금) → 확장(수출입) 단계화 제안 여지.
3. **라벨 데이터가 관건** — 지도학습 활용을 명시했으므로, 최종 판정(Label) 데이터의 축적 기간·표본 수·판정 일관성이 기간·성능을 좌우. 선행 준비사항으로 강조 필요.
4. **차세대 일정 종속성** — 데이터 구조 안정화 이후 AI 고도화라는 인식이 명시되어 있어, 전체 일정이 당행 차세대 일정에 종속될 수 있음. 회신 시 전제조건으로 명기.
5. **파일럿 2~3개월은 문의자 측 자체 추정** — 회신의 단계별 기간과 정합성 있게 제시.
